Uppgift 3

Uppgift 1:

egrep "[a-z]{4,5}[0-9]{2,3}" filnamn
Jag använder mig av egrep för att använda regex tillsammans med grep. Mitt regex uttryck letar efter alla små bokstäver mellan a-z och endast då det är fyra eller fem av dessa i följd. Följt av alla siffror två eller tre gånger. Detta uttryck hittar då liu id.n.

Uppgift 2:

egrep "[0-9]{4}\-[0-9]{2}\-[0-9]{2}" filnamn
För att hitta datum i en text består mitt regex uttryck av: alla siffror 4 gånger vilket hittar året, följt av ett bindestreck där jag använder backslash för att förtydliga att jag letar efter ett bindestreck eftersom bindestreck kan ha en annan betydelse i regex. Efter bindestrecket letar jag efter månaden i form av två siffror, sedan kommer ett till bindestreck. Efter bindestrecket letar jag efter dagarna i samma form som månaden, två siffror.

Uppgift 3:

egrep "[0-9]{4}\-(0[1-9]|1[0-2])\-(0[1-9]|1[0-9]|2[0-9]|3[01])"
Detta är då ett mer utvecklar regex uttryck som ser till att månaderna och dagarna faktiskt finns. Uttrycket för året är detsamma, månaden ser till att ifall månaden börjar på noll så kan följande siffra vara siffrorna mellan ett och nio. Men om månaden börjar på ett så kan följande siffra endast vara ett eller två. För att skapa detta "antingen eller" uttryck använder jag en pipe mellan uttrycken för att förklara att det är antingen eller. Uttrycket för dagarna är uppbygt på samma sätt, börjar dagen med noll så kan följande siffra endast vara siffrorna ett till nio. Ett eller två så kan följande siffra vara vilken som helst. Börjar dagen med tre så kan följande siffra endast vara noll eller ett.

Uppgift 1:

find / "*.py" -executable -exec grep -H '#! /usr/bin/env python3' {} \;
Jag använder find funktionen för att söka efter filer, sökvägen sätter jag till / för att söka igenom hela datorn. Argumentet för vad jag söker efter är är inom dubbla citatecken, jag letar efter alla filer som slutar på .py. * före .py berättar att det är okej att det står text innan .py. Sedan använder jag flaggan -executable för att endast matcha de filer som är körbara för mig. Flaggan -exec används för att jag ska kunna kombinera find med andra metoder, i detta fall grep funktionen. Grep letar igenom alla matchningar efter argumentet inom enkla citatecken, jag valde enkla citatecken just för att dom säger att argumentet ska se ut precis som argumentet är skrivet. Flaggan -H gör att när filer som matchar hittas skrivs detta ut i terminalen, utan -H står det i princip bara mitt grep uttryck flera gånger. {} används som en sorts behållare, alla filer jag hittar med find sparas i behållaren så grep funktionen kan utföras på dessa filer. \; avslutar -exec.

2.6 Smått och gott

find ~ -executable
find används för att söka efter filer och kataloger. ~ tecknet representerar min hemkatalog och säger att det är där find ska söka efter filer. Flaggan -executable säger att find endast ska söka efter de filer som är körbara.
find ~ "*.py" -executable -exec grep -H '#! /usr/bin/env python3' {} \;
Nu letar jag endast efter python filer ("*.py") som är körbara (-executable). För att veta om filerna innehåller #! /usr/bin/env python3 vilket gör dom körbara som ett program, använder jag flaggan -exec för att kombinera find med metoden grep som letar igenom filerna efter argumentet. Flaggan -H gör att när en fil innehåller argumentet skrivs den ut i terminalen. {} dessa måsvingar används som en behållare som sparar de filerna som matchat med find och -executable så grep kan leta igenom dem. \; avslutar -exec.
grep '^#' ~/*
grep letar igenom alla filer efter det inmatade argumentet som jag har inom citattecken. Eftersom alla kommentarer i python börjar med # så sätter jag ^ innan vilket innebär att grep endast kommer matcha med rader som börjar med #. ~ är sökvägen där jag börjar min sökning, /* betyder att grep ska söka i alla filer som ligger direkt under min hemkatalog men inte djupare.
